# 前端设计稿（Fluent 2 / 浅蓝主题）

> 适用范围：SCP-cv 前端控制台重构稿。本稿描述布局、信息架构与操作逻辑，不规定具体技术栈实现。
> 设计语言：对齐 [Microsoft Fluent 2 Design](https://fluent2.microsoft.design/)，并严格遵循仓库根目录的 [`DESIGN.md`](../DESIGN.md)（视觉规范、交互规范、可访问性、设计到开发映射）。
> 文档冲突时，处理顺序：用户当前明确要求 → `DESIGN.md` → 本设计稿 → Fluent 2 官方约定 → 通用工程实践。

---

## 1. 设计目标与原则

### 1.1 项目目标

| 目标 | 说明 |
|------|------|
| 操作可达性 | 现场人员单手单屏即可完成「调用预案 → 切换源 → 控制播放 → 切换大屏模式」全链路 |
| 状态透明 | 四个窗口的播放状态、SSE 连接状态、系统音量与电源指令必须长时可见 |
| 错误可恢复 | 所有 REST 失败、SSE 断连、设备 TCP 失败都应给出明确反馈与重试入口 |
| 风格一致 | 全站遵循 `DESIGN.md`：语义化 token + 浅蓝 Brand + 克制阴影与渐变 |
| 适配现场 | 浅色主题为默认、键盘友好、对 1280–2560 宽屏与 13″ 触屏一致体验，竖屏移动端可用 |

### 1.2 设计原则（与 `DESIGN.md §2` 对齐）

| 原则 | 在本设计稿的体现 |
|------|------------------|
| **清晰优先** | 每页标题表达当前任务；主操作单一、不与同强度按钮并列；错误说明原因+修复方式；保存中/成功/失败状态可见 |
| **一致性优先** | 同一语义复用同一组件（如音量统一 Slider + 静音 Switch）；同一状态颜色/图标/文案一致；同一层级字号、间距、密度统一 |
| **效率优先** | 高频操作主路径可见（首页直接给电源/音量/大屏模式）；低频/危险下沉（应急放 TitleBar Flyout）；表单合理默认 + 即时校验 |
| **包容性优先** | 鼠标可达即键盘可达；不只用颜色传达状态（必配文本/图标）；交互元素可聚焦；触控目标 ≥ 44 × 44 px |
| **克制表达** | 大面积色块仅用于品牌强调；阴影按 `shadow.{2,4,8,16,28}` 分级，不滥用；Brand 色不用于装饰，专门留给主操作和选中态 |

### 1.3 页面类型映射（与 `DESIGN.md §3.2` 对齐）

| 本设计稿页面 | DESIGN.md 类型 | 关键模板 |
|--------------|---------------|----------|
| `/dashboard` | Dashboard | 指标/状态卡 + 快捷指令 |
| `/sources` | List | 标题、搜索、筛选、卡片/表格、空态 |
| `/display/:target` | Detail（单对象+控制） | 标题区、元信息、分区控制、相关操作 |
| `/ppt-focus/:windowId` | Detail（全屏） | 演讲者侧专用沉浸视图 |
| `/scenarios` | List + Form/Workflow | 列表浏览 + 多步表单（移动端） |
| `/settings` | Settings | 顶部 AppHeader + Tab 分组 |
| 任意页面错误/空态 | Empty / Error | 图标 + 描述 + 恢复操作 |

## 2. 设计令牌（Design Tokens）

> 严格遵循 `DESIGN.md §5` 的两层 token 模型：**alias token**（语义角色，业务代码必须用）+ **global token**（原始 HEX/PX 值，仅 token 文件用）。本节给出 alias 命名与项目落地的 global 值；业务代码不得直接写 HEX。
> 命名格式：`<category>.<role>.<state?>.<emphasis?>`。

#### 状态（`color.status.{info,success,warning,error}.*`）

| 状态 | foreground | background | 用途 |
|------|-----------|------------|------|
| Info | `#0F6CBD` | `#EAF4FE` | 普通信息提示、默认 Tag |
| Success | `#107C10` | `#DFF6DD` | 完成、播放中、上传成功 |
| Warning | `#9D5D00` | `#FFF4CE` | loading、暂停、SSE 重连中 |

* **导航形态**：左侧 NavigationView。≥ 1280 px 全展（240 px），1024–1280 px 改为 Compact（48 px，仅图标），< 1024 px 收为 Minimal（顶部汉堡 + 抽屉）。

## 4. 各页面设计

> 共用规范：每页主区使用「12 栅格 × `spacing.l` (16) gutter × `spacing.2xl` (24) 外边距」。卡片以 `background.card` + `radius.large` (8) + `border.subtle` 呈现，默认无阴影；Header 用 H2 + 右侧操作区。

### 4.1 仪表盘 `/dashboard`

> **页面类型**：Dashboard（指标 + 快捷入口） · **状态覆盖**：idle / loading（卡 Skeleton）/ error（设备 TCP 失败 MessageBar） · **无 empty 态**（运行态恒有值）

**定位**：运行现场的「开机/关机 + 总控」面板。仅承载**无需进入子页即可完成**的顶层操作：查看运行态、切大屏模式、改系统音量、下达设备电源指令。预案调用、上传、窗口状态等明细能力统一下沉到对应子页。

```
╔════════════════════ Hero (Brand 渐变 → 浅蓝) ══════════════════════╗
║ Eyebrow  COMMAND CENTER                                            ║
║ H1       双屏模式 · SSE 已连接                                      ║
║ Caption  首页用于全局运行态与顶层指令。媒体源/预案/显示控制在侧栏    ║
╚═════════════════════════════════════════════════════════════════════╝

┌── 大屏模式 (4 col) ────┐ ┌── 系统音量 (4 col) ──┐ ┌── 电源指令 (4 col) ──┐
│ Eyebrow  BIG SCREEN    │ │ Eyebrow  SYSTEM VOL  │ │ Eyebrow  POWER       │
│                        │ │                      │ │                      │
│  ◉ 单屏    ○ 双屏      │ │ ──●──────── 65       │ │ 拼接屏  ⏻ 开 · ⏻ 关  │
│  Caption：单屏下窗口 2 │ │ [ 系统静音切换 ]     │ │ 电视左  ⇆ 切换       │
│  自动静音              │ │ 后端：windows_core   │ │ 电视右  ⇆ 切换       │
└────────────────────────┘ └──────────────────────┘ └──────────────────────┘
```

**操作逻辑**

1. **Hero** 只做信息展示：大屏模式（`runtime.big_screen_mode`）、SSE 连接状态、简短说明。不放操作控件，避免与下方三张卡重复。
2. **大屏模式卡**：SegmentedControl 单选，即点即调 `PATCH /api/runtime`；切换成功后 NavigationView 的「大屏」分组实时刷新（单屏仅「大屏」一项；双屏展开为「大屏左 / 大屏右」）。
3. **系统音量卡**：`Slider 0–100` + `Toggle 静音`，对应 `PATCH /api/volume`。`后端` 一行显示 `volume.backend`（`windows_core_audio` / `runtime_state`），当回退到 `runtime_state` 时文字变 Warning。
4. **电源指令卡**：
   - 拼接屏：`开机` 调 `POST /api/devices/splice_screen/power/on/`，`关机` 为 Danger 按钮并走 Dialog 二次确认，提示「发送开/关机 TCP 指令至 192.168.5.10:8889，需等待 5 s 第二帧」；
   - 电视左/右：`⇆ 切换` 调 `POST /api/devices/tv_left|tv_right/toggle/`，说明该指令不读取真实开关状态，仅发送切换。
   - 发送失败（`device_error`）在卡片底部 Inline MessageBar 行内报错 + 重试。
5. **应急操作**不出现在首页主体，统一放 TitleBar 右侧的「⚠ 应急」Flyout：`重置全部窗口` / `关闭播放器`。

> 首页刻意不再承载预案调用、文件上传、窗口状态列表。这些能力分别由 `/scenarios`、`/sources`、`/display/*` 承接，避免首页承担过多职责。

* **离线直播源**：离线超过 1 小时会被后端自动清理；未达 1 小时的以灰色 Caption 提示「已离线 XX 分钟」，便于现场判断是否重推。

**窗格内容规则**

| 源类型 / 状态 | 预览区 | 文本区 |
|---------------|--------|--------|
| 「保持」（`source_state=unset`） | `background.brandSelected` 底 + 图标 `Dismiss24Regular` | 「保持当前窗口状态」 |
| 「黑屏」（`source_state=empty`） | 纯黑占位 | 「切换后窗口黑屏」 |

* **编辑**：打开「编辑覆盖卡片」（见 4.5.3），抽屉保持在下方备用但被卡片遮盖。
* **置顶/取消置顶**：调 `POST /api/scenarios/{id}/pin/`。
* **删除**：Dialog 二次确认后调 `DELETE /api/scenarios/{id}/`，删除后自动关闭抽屉。

  * `保持` → 「切换到本预案时不改变该窗口内容」

* `切换` 状态：依次展开

     - 选中后出现 `流标识` 输入框（必填），保存时前端调 `POST /api/sources/web/` 的直播兄弟接口（或约定 `srt_stream` 新建流程）生成对应直播源并带回 `source_id`；
     - 若现有源已经是直播源，流标识可以留空，继续使用该源既有标识。
  2. **窗口音量**：`Slider 0–100`（图片类型源时整体 disabled，label 显示「图片不支持音量调整」）。
  3. **循环播放**：仅 `video` / `audio` 源显示的 `Switch`。
* 源类型与可编辑项映射：

| `source_type` | 源选择 | 窗口音量 | 循环播放 |
|---------------|--------|----------|----------|
| `ppt` | ✓ | ✓ | ✗ |
| `video` | ✓ | ✓ | ✓ |
| `audio` | ✓ | ✓ | ✓ |
| `image` | ✓ | ✗（灰化） | ✗ |
| `web` | ✓ | ✗（灰化） | ✗ |
| `srt_stream` / `rtsp_stream` / `custom_stream`（直播） | ✓ + 流标识 | ✓ | ✗ |

**大屏模式与窗格布局联动**

* 顶部 Toolbar 切换到 `单屏` → 大屏左 / 大屏右两格实时合并为一格「大屏」，右上角 SegmentedControl 沿用左格状态。
* 顶部 Toolbar 切换到 `双屏` → 大屏拆回两格，之前合并时的配置自动复制到大屏左，大屏右重置为 `保持`。
* 顶部 Toolbar 选 `保持`：下方田字按当前 `runtime.big_screen_mode` 渲染布局，但**提交时 `big_screen_mode_state = unset`**，不会改大屏模式。

**底部按钮**

* `取消` → 关闭覆盖卡片，不保存；如有未保存改动，弹 Dialog「放弃修改？」。
* `保存预案` → 根据入口分流：
  * 新建 / 编辑 → `POST` 或 `PATCH /api/scenarios/{id}/`；
  * 从当前状态生成 → 同 `POST /api/scenarios/`，**不再单独调 `POST /api/scenarios/capture/`**，前端直接按表单生成 `scenario` JSON 提交，这样「从当前状态生成」的结果和「新建」完全一致，只是初始填写不同，便于用户保存前继续微调。

**校验**

* 预案名称必填；长度 ≤ 32。
* 任一窗格状态为 `切换` 时，`source_id` 必选；直播源必填流标识（若新建）。
* 提交前对 `source_state = set` 且 `source_id` 指向已不可用的源作行内警告，允许继续保存但提示「该源当前不可用」。

### 5.1 Button 按钮（对齐 `DESIGN.md §12.2`）

按钮**类型**：

| Primary | `background.brand` 实心 + `text.inverse` | 唯一主操作：保存预案、激活预案、上传并保存、应用显示器 |
| Secondary | 透明底 + `border.default` + `text.primary` | 次操作：取消、关闭显示、上一步、上传但不保存 |
| Subtle | 透明底 + `text.brand`，Hover 微底 | 工具栏低强调：刷新、编辑、查看 |
| Transparent | 透明底 + `text.secondary`，无边框 | 极低强调，紧凑区域（Tab 上下文菜单触发） |
| Danger | `status.error.background` 边框 + `status.error.foreground` 文字 | 关机、删除、关闭显示 |
| Icon Button | 仅图标 + 必带 `aria-label` | 关闭按钮、`⋯` Flyout、Pin、应急 Flyout 触发 |

**尺寸**（对齐 `DESIGN.md §4.1` 的 4 px 网格）：

| 断点 | 高 | padding | 圆角 |
|------|----|---------|------|
| 桌面（紧凑） | 28 px | 4 px / 12 px | `radius.medium` (4) |
| 桌面（默认） | 32 px | 6 px / 16 px | `radius.medium` (4) |
| 桌面（大） | 40 px | 10 px / 20 px | `radius.medium` (4) |
| 移动 sm/xs | 48 px + 100 % 宽 | 12 px / 16 px | `radius.medium` (4) |

**状态规则**：

* 主按钮文案使用动宾结构：「保存预案」「上传并保存」。
* 危险按钮文案明确后果：「删除预案」「拼接屏关机」。
* `loading` 状态保留按钮宽度避免抖动，禁用重复提交，左侧叠加 12 px Spinner。
* `disabled` 透明度 0.4，光标 `not-allowed`，所有 hover/focus 效果失效。
* Icon Button 必须带 `aria-label`，必要时配 Tooltip。
* Toggle 行为不再单独列为按钮类型，统一交给 **Switch** 或 **SegmentedControl**。

### 5.2 Input 输入框（对齐 `DESIGN.md §12.3`）

* 结构：Label + Input(可加 prefix/suffix) + Hint/Error。
* 高 32 px / 圆角 `radius.medium` (4) / 边框 `border.default`。
* Focus：边框变 `border.focus` 并叠 2 px Focus 环（`box-shadow: 0 0 0 2px color.border.focus`）。
* Error：边框变 `border.error`，Hint 区文字变 `text.error` 并配 `ErrorCircle16` 图标。
* `placeholder` 不替代 Label；必填项不仅依赖红色星号，必须有「（必填）」文本或同等表达。
* 移动端高 44 px，Label 12 px caption + 上间距 4 px。

### 5.3 Select / Combobox（对齐 `DESIGN.md §12.4`）

* 选项 < 5 且短，改用 Radio / SegmentedControl，不强用 Select。
* 选项 ≥ 10 时必须支持搜索。
* 多选展示已选数量并支持「清除」。
* 下拉浮层用 `shadow.4`，圆角 `radius.large` (8)。
* 不在下拉中嵌入复杂表单（如「新增源」必须单独走 Drawer/Sheet）。

### 5.4 Checkbox / Radio / Switch（对齐 `DESIGN.md §12.5–12.7`）

| 控件 | 用法 |
|------|------|
| Checkbox | 多选或开关式确认；批量列表头支持 indeterminate；标签描述选中后状态 |
| Radio | 2–5 项互斥选项；默认应有选中值 |
| Switch | 即时生效的二元状态：「循环播放」「静音」「窗口静音」「保留进度」（如未来引入）|

* Switch 不用于需提交后才生效的设置；如有复杂后果（如「关闭 SSE 重连」）必须配说明或确认。
* Switch Label 描述开启状态：「启用循环播放」。

### 5.5 SegmentedControl（项目内统一控件）

* 用于 2–4 个互斥选项即时切换：大屏模式（保持/单/双）、源状态（保持/黑屏/切换）、显示控制窗口选择（W1/W2/W3/W4）。
* 选中态：`background.brandSelected` 底 + `text.brand` 文字 + 内含 1 px brand 描边。
* 触控目标 ≥ 44 × 44 px；移动端等宽分布。

### 5.6 Card 卡片（对齐 `DESIGN.md §12.9`）

* 容器层级：`background.canvas` → `background.card` → 嵌套子卡 `background.subtle`。
* 默认无阴影，靠 `border.subtle` 区分；Hover 卡片可用 `shadow.2`。
* 卡片整体可点击时（如预案卡），内部不再放独立点击目标，附属操作收到 ActionSheet。
* 同组卡片必须高度、内容结构、操作位置一致。
* Padding：桌面 16 px (`spacing.l`)，嵌套 12 px (`spacing.m`)；移动 16 px / 12 px。

### 5.7 Tag / Chip / Badge

| 状态 | foreground / background |
|------|--------------------------|
| `playing` | `status.success.foreground` / `status.success.background` |
| `idle` / `stopped` | `text.tertiary` / `background.subtle` |
| `loading` | `status.warning.foreground` / `status.warning.background`，左侧 4 px 旋转点 |
| `paused` | `text.secondary` / 透明底 + `border.default` |
| `error` | `status.error.foreground` / `status.error.background` |
| `Pinned` | `text.inverse` / `background.brand` + 星形图标 |

* padding 4 px / 12 px，圆角 `radius.small` (2)，字号 `type.caption1`。
* Tag 不作为筛选触发器（避免误触），筛选交给类型 Pills 与搜索框。

### 5.8 Dialog（对齐 `DESIGN.md §12.10`）

* 必备：标题（任务/后果）+ 正文上下文 + 主按钮（执行）+ 次按钮（取消）。
* 危险确认必须用 Danger 主按钮，标题写明对象与影响范围（如「删除预案：早会方案？此操作不可恢复」）。
* 焦点：打开时移入 Dialog 内合理位置；关闭后回到触发元素。
* `Esc` 仅关闭非危险 Dialog。

### 5.9 Drawer / Sheet（对齐 `DESIGN.md §12.11`）

* 桌面：右侧 Drawer 用于详情/编辑（媒体源添加、预案预览）；左侧 Drawer 仅用于 md 断点的导航折叠。
* 移动 sm/xs：所有右侧 Drawer 替换为**底部上拉 Sheet**，顶部带 4 × 32 px 拖动手柄（`radius.circular`）。
* Drawer 内长表单必须有底部固定操作栏；不要在 Drawer 内打开复杂二级 Drawer。

### 5.10 Tooltip（对齐 `DESIGN.md §12.12`）

* 仅放短说明，最大宽 240 px，6 / 12 px padding。
* 不放关键流程信息或交互控件。
* **移动端**：Tooltip 失效，必须为图标按钮提供文字 Label 或长按 Sheet 替代。
* 显示延迟 150 ms，隐藏立即。

### 5.11 Toast（对齐 `DESIGN.md §12.13`）

* 用于非阻断的全局反馈。
* 成功 Toast 自动消失（3.2 s）；错误 Toast 不自动消失，必须可手动关闭并提供「重试」入口。
* 关键错误不能只用 Toast，页面内也应保留错误状态。
* 同时最多展示 3 条；超过堆叠时新条替换最早条。
* **移动端 sm/xs 改为顶部居中**，避开底部 TabBar；带 `safe-area-inset-top`。

### 5.12 Message Bar（对齐 `DESIGN.md §12.14`）

* 放在与问题最相关的位置（如表单字段下方、页面顶部、卡片底部）。
* 错误消息条说明修复方式。
* 可关闭后不影响完成任务。
* 用于：SSE 断开（页面顶部）、上传失败（卡片底部）、设备 TCP 失败（电源卡片底部）。

### 5.13 Tabs / Pivot（对齐 `DESIGN.md §12.15`）

* 不超过 7 项；超过考虑导航或筛选。
* Tab 切换不丢未保存内容；必要时 Dialog 提示。
* 当前 Tab 必须有明确选中态（Brand 下划线 + Brand 文字）。
* 键盘：`←/→` 切焦点，`Enter` 激活；移动端支持手势滑动。

### 5.14 Breadcrumb（对齐 `DESIGN.md §12.16`）

* 仅在二级页（PPT 专注模式、预案编辑大卡片）使用；浅层页面靠页面标题区即可。
* 末项为当前页，不可点击或弱化。
* 不替代页面标题。

### 5.15 Menu / Flyout（对齐 `DESIGN.md §12.17`）

* 收纳命令：列表行末 `⋯` 菜单（打开到窗口、下载、删除）、应急 Flyout（重置全部、关闭播放器）。
* 菜单项使用动词或明确名词。
* 危险项分组并使用 `status.error.foreground` 文字。
* 禁用项必须配 Tooltip 说明不可用原因。
* 键盘：方向键导航、`Enter` 触发、`Esc` 关闭。

### 5.16 Progress

* 上传进度：`ProgressBar` 高 4 px，Brand 轨道，右侧百分比 caption；`shadow.none`。
* PPT 翻页进度：同 ProgressBar 但只展示，不可点。
* 长任务（> 1 s）：Skeleton；长任务（> 3 s）配明确「正在…，可后台继续」。

## 6. 状态与异常（对齐 `DESIGN.md §13`）

### 6.1 状态模型

所有异步区域（页面、卡片、列表、表单提交）至少覆盖：

```
idle → loading → success
              → empty
              → error
```

每个页面验收时按此清单逐一检查；缺任一态视为不完整。

### 6.2 Loading

| 场景 | 推荐模式 |
|------|---------|
| < 300 ms | 不展示加载态，避免闪烁 |
| 300 ms – 1 s | 局部 Spinner / 按钮 loading（保留按钮宽度） |
| > 1 s | Skeleton 保留布局稳定 |
| 长任务 / 上传 | ProgressBar + 可取消 / 后台说明 |

* 不使用整页遮罩；首页/列表/Drawer 各自局部 loading。
* 表格用 Skeleton 行；卡片用 Skeleton 头 + 行。

### 6.3 Empty 空状态

包含三要素：当前没什么 + 为什么没有 + 用户可做什么。

```
暂无 PPT 源
上传或注册 PPT 源后，可在这里浏览、打开到窗口或加入预案。
[+ 上传 PPT]
```

### 6.4 Error 错误状态

```
无法加载预案列表
网络连接不稳定或服务器暂时不可用，请稍后重试。
[重试]
错误代码：SCENARIO_LIST_FETCH_FAILED
```

* 含问题描述 + 可能原因 + 修复动作 + 必要时错误代码。
* 后端结构化错误：`{ detail, code }`，前端按 `code` 给中文话术；`detail` 仅在开发模式露出。

### 6.5 Confirm vs Undo（对齐 `DESIGN.md §13.6 / §13.7`）

| 操作 | 必须 Confirm | 可 Undo（4 s 内） |
|------|--------------|--------------------|
| 删除预案 | ✓（Dialog） | — |
| 拼接屏关机 | ✓（Dialog，写明 TCP 等待 5 s） | — |
| 关闭显示（窗口） | — | Undo Toast「已关闭显示 [撤销]」 |
| 移动/归档（未来） | — | Undo Toast |
| 上传源 | — | Undo「已上传 [撤销]」 |
| 切换大屏模式 | — | Undo「已切换为单屏 [撤销]」 |
| 系统关机 / 关闭播放器 | ✓（Dialog） | — |

确认文案必须具体说明对象、范围、后果。

### 6.6 项目级异常表

| 场景 | 表达 |
|------|------|
| SSE 断开 | TitleBar Tag 变 Warning「SSE 重连中」+ 顶部 Banner，含「立即重试」 |
| REST 401/403 | Dialog「登录已过期 / 无权限」+ 跳转处理 |
| REST 5xx | Toast Error + 错误码 + 「重试」（可重试） |
| 大屏模式与窗口冲突（Single 进入大屏右） | EmptyState「Single 模式下隐藏」+ 主按钮「切换到 Double」 |
| 设备 TCP 失败 | 行内 MessageBar (Error) + 显示目标 IP/端口 + 「重试」按钮 |
| 上传文件超过限制 / 类型不支持 | Inline MessageBar（在文件选择下方），保留用户已填的「显示名称」 |
| 直播源离线 < 1 h | 列表项灰色 caption「已离线 XX 分钟」 |
| 直播源离线 ≥ 1 h | 后端自动清理；前端 Toast「直播源 XXX 已自动移除」 |

## 7. 响应式断点（对齐 `DESIGN.md §17`）

### 7.1 断点定义

| Token | 范围 | 设备 | NavPane / 主导航 | 内容栅格 | Drawer / Dialog |
|-------|------|------|------------------|----------|------------------|
| `breakpoint.2xl` | ≥ 1920 px | 超宽屏 / 控制中心 | 全展 NavPane 240 px | 12 列 / gutter 16 / 内容 max 1440 px | 右侧 480 px Drawer |
| `breakpoint.xl` | 1440–1919 px | 大屏桌面 | 全展 240 px | 12 列 / gutter 16 / 内容 max 1280 px | 右侧 480 px Drawer |
| `breakpoint.lg` | 1024–1439 px | 常规桌面 | 全展 200 px | 12 列 / gutter 16 | 右侧 440 px Drawer |
| `breakpoint.md` | 768–1023 px | 平板、小窗口 | Compact 48 px（仅图标）+ Tooltip | 8 列 / gutter 16 | 居中 Dialog；左右双栏页面切上下双栏 |
| `breakpoint.sm` | 480–767 px | 常规手机（含竖屏与小平板竖屏） | **底部 TabBar（5 项 + 「更多」）** + 顶部精简栏 | 4 列 / gutter 12 / 外边距 16 | 底部上拉 Sheet（取代 Drawer） |
| `breakpoint.xs` | < 480 px | 小屏手机 / 折叠屏内屏窄态 | 同 sm，但隐藏顶部副标题与 caption | 4 列 / gutter 8 / 外边距 12 | 上拉 Sheet 全高 |

> 与 DESIGN.md 一致：xs/sm/md/lg/xl/2xl 命名、`<480 / 480–767 / 768–1023 / 1024–1439 / 1440–1919 / ≥1920` 范围。下文所有页面的「桌面形态」泛指 `≥ md` 即 ≥ 768 px；「移动端竖屏」指 `xs / sm` 即 < 768 px。

> **PPT 专注模式**仅在 ≥ 1024 px (`lg+`) 横屏可用；其它情况打开 `/ppt-focus/:windowId` 时显示 EmptyState：「请使用更大屏幕或横屏后再进入专注模式」+ 主按钮「返回屏幕控制」。

### 7.2 移动端 Shell（sm / xs，竖屏主形态）

```
┌──────────────────────────────────────┐
│ TopBar (48 px)                       │
│  ☰  SCP-cv          ⚠  ⓘ            │  ← 左：抽屉触发；右：应急 Flyout / 帮助
│ Caption: 双屏 · SSE 已连接            │  ← 第二行 12 px caption（xs 隐藏）
├──────────────────────────────────────┤
│                                      │
│ Page Content                         │
│ （单列纵向，sticky Sub-Toolbar）      │
│                                      │
│                                      │
├──────────────────────────────────────┤
│ Bottom TabBar (56 px, sticky)        │
│  🏠首页  🖥显控  📚源  🗂预案  ⋯更多 │
└──────────────────────────────────────┘
```

* **底部 TabBar** 5 个固定项（Fluent 「Tab Bar」模式）：`首页 / 显控 / 媒体源 / 预案 / 更多`。
  - `显控` 默认指向 `/display/big-left`；二级选择器（窗口 1/2/3/4）以顶部 SegmentedControl 形式出现在内容区头部；
  - `更多` 弹底部 Sheet 列表：`设置`、显示模式切换、应急、版本号。
* **顶栏汉堡菜单**仅在用户希望直接跳转到平台级隐藏入口（如 `/about` 重定向、日志）时使用，正常导航全部走底部 TabBar，避免现场单手到不了顶。
* **TitleBar 内容**收敛：仅保留品牌、应急 Flyout、帮助；大屏模式 / SSE 状态以 caption 形式显示在第二行（xs 断点隐藏，进设置查看）。
* **应急 Flyout** 在 sm 断点直接占满底部 1/2 高度的 Sheet。

### 7.3 触控规约

* 所有可点击目标 **最小命中区域 44 × 44 px**；视觉尺寸可小，但 `padding` 撑足。
* `Slider` 拇指放大到 24 px；`Switch` 高度 28 px；`Combobox` 行高 44 px。
* `Tag / Chip` 仅作展示；不再让用户点 Tag 进行筛选（避免误触），筛选统一交给类型 Pill / 搜索框。
* 主操作按钮在 sm 断点占满父容器宽度（`width: 100%`），并固定 48 px 高；次操作按钮放下方一行，避免横向并排误触。
* **手势**：
  - Sheet / Drawer：下滑手柄关闭；
  - 列表：下拉刷新（带 Brand 进度环）；
  - Pivot Tab：左右滑动切换（仅在内容区无横向滚动时启用）。
* `hover` 仅作 `:focus-visible` 兜底；不依赖 `:hover` 表达状态。

## 8. 可访问性 (A11y) 与移动端体验

* 所有交互元素 Focus 环：`outline: 2px solid var(--color-border-focus); outline-offset: 2px`。
* 颜色对比：正文 ≥ 4.5:1，大标题 ≥ 3:1（Brand Primary 在白底下满足正文要求）。
* 键盘：左侧 Nav `Tab + Enter`、Pivot `←/→`、Slider `←/→` 步长 1 / `Shift+←` 步长 10；不为 PPT 专注模式额外定义快捷键，沿用按钮的默认 Tab + Enter / Space 触发即可。
* 错误反馈：`role="alert" + aria-live="assertive"`。
* PPT 专注模式提供「OS 全屏」与「降低明度」两个无障碍开关。
* **触控目标 ≥ 44 × 44 px**，sm / xs 断点对所有可点击元素的 `padding` 撑足。
* **方向监测**：监听 `matchMedia('(orientation: portrait)')` 与窗口尺寸；进入 PPT 专注模式或 ≥ 1024 px 横屏专属页（如 Display 双栏）时，竖屏显示「请旋转或返回」提示。
* **安全区**：Sticky 顶/底栏需 `padding-top: env(safe-area-inset-top)` / `padding-bottom: env(safe-area-inset-bottom)`，兼容刘海屏与 Home 横条。
* **iOS Safari** 的 100 vh 问题用 `100dvh` 或 `--app-height` JS 变量兜底，确保底部 TabBar 不被遮挡。

## 10. 文案规范（对齐 `DESIGN.md §19`）

### 10.1 文案原则

- **具体**：说明用户正在做什么。
- **简短**：去掉不影响理解的词。
- **可执行**：错误和空状态应给下一步。
- **一致**：同一概念使用同一名称（统一「源」而非源/媒体源/Source 混用；统一「窗口」而非屏幕/输出/通道）。
- **中性**：避免责备用户。

### 10.2 项目级常用文案模板

| 场景 | 模板 |
|------|------|
| 按钮 | 动宾结构：`保存预案` / `上传并保存` / `上传但不保存` / `激活预案` / `关闭显示` / `重试` / `取消` |
| 删除确认 | `删除预案：早会方案？此操作不可恢复，预案中四窗口配置会被一并移除。` |
| 关机确认 | `拼接屏关机？发送关机 TCP 指令至 192.168.5.10:8889，5 秒后再发送第二帧；此过程不可中断。` |
| 空态 - 媒体源 | `暂无 [PPT/视频/图片] 源。上传或注册后，可在这里浏览、打开到窗口或加入预案。[+ 上传]` |
| 空态 - 预案 | `暂无预案。从当前播放状态生成预案，或手动创建第一个预案。[+ 新建预案] [▽ 从当前状态生成]` |
| 错误 - 加载失败 | `无法加载预案列表。网络连接不稳定或服务器暂时不可用，请稍后重试。[重试] 错误代码：SCENARIO_LIST_FETCH_FAILED` |
| 错误 - 字段校验 | `预案名称不能为空，请输入 1–32 个字符。` |
| 错误 - TCP 失败 | `拼接屏关机指令发送失败（TCP 连接超时 192.168.5.10:8889）。请检查网线与设备电源。[重试]` |
| Undo Toast | `已切换为单屏。 [撤销]` |

### 10.3 禁用文案

避免使用：「出错了」「非法参数」「请正确填写」「操作失败」「未知错误」。除非作为兜底且配合错误代码与重试入口。

## 12. 验收清单

### 12.1 通用（对齐 `DESIGN.md §23.1`）

- [ ] 页面主任务清晰，单页只有一个视觉主标题
- [ ] 颜色、字号、间距、圆角、阴影全部走 alias token，无 HEX/px 硬编码
- [ ] 浅蓝 Brand 仅用于 Primary 按钮、激活态、链接、Focus 环；不用于装饰
- [ ] 默认卡片以 `border.subtle` 区分而非阴影；阴影按 `shadow.{2,4,8,16,28}` 分级
- [ ] 所有异步区域覆盖 idle / loading / success / empty / error 五态
- [ ] 表单字段与 Label 关联；错误紧贴字段；必填项不只用红星表达
- [ ] 危险操作必须 Confirm（Dialog）+ Danger 主按钮 + 写明对象/范围/后果；可恢复操作优先 Undo Toast 而非阻断 Confirm
- [ ] `single` 模式下 NavigationView 自动隐藏「大屏右」入口
- [ ] WCAG 2.1 AA：正文 ≥ 4.5:1，大标题 ≥ 3:1；不仅依赖颜色传达状态
- [ ] 键盘可达：Tab + Enter / Space 触发；Dialog 焦点循环；关闭后焦点回触发元素
- [ ] 支持 `prefers-reduced-motion`

### 12.2 桌面（≥ md，768 px+）

- [ ] SSE 状态、运行模式、系统音量在 TitleBar 长时可见
- [ ] 仪表盘只有 Hero + 大屏模式 + 系统音量 + 电源指令四块内容，无预案/上传/窗口状态

- [ ] 显示控制页 PPT 专注链接、上传按钮文案与本稿一致（「上传但不保存」「上传并保存」）
