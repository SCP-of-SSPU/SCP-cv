# SCP-cv 使用文档

## 1. 系统概述

SCP-cv 是一套基于 Vue + Django + PySide6 的单机大屏内容播控系统，能够在本地主机上完成：

- SRT 视频流的接收与低延迟全屏播放（OBS → SRT → MediaMTX → SRT → libVLC）
- 多窗口独立播放与左右拼接双屏显示（4 路输出窗口，每窗口独立会话）
- 通过 Vue Web 控制台（REST + SSE）和外部系统（gRPC）两种方式操控

系统由三个主要进程协作：**Vue 前端**（控制台 UI）、**Django 服务端**（REST API + gRPC）和 **PySide6 播放窗口**（libVLC 渲染引擎），后端与播放器通过 SQLite 数据库同步状态。播放控制统一在浏览器 Web 控制台完成，本地 PySide 仅保留启动器和播放窗口。

## 2. 环境准备

### 2.1 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 |
| Python | 3.12 或更高版本 |
| Node.js | 20 或更高版本 |
| 显示器 | 至少 1 台（拼接模式需 2 台） |

### 2.2 安装 uv

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

### 2.3 安装项目 Python 与同步依赖

```powershell
cd E:\Projects\SCP-cv

# 安装项目固定的 Python 版本（遵循 .python-version）
uv python install

# 同步项目依赖（默认包含 dev 依赖组）
uv sync
```

> 项目已切换到 `uv` 工作流；根目录的 `requirements*.txt` 仅作为历史兼容清单保留，不再作为推荐安装入口。仅需运行时依赖时，可改用 `uv sync --no-dev`。

### 2.4 配置环境变量

复制 `.env.example` 为 `.env`，按需修改：

```env
# Django 基础配置
DJANGO_DEBUG=False
DJANGO_SECRET_KEY=请替换为随机密钥
DJANGO_ALLOWED_HOSTS=*

# gRPC 服务端口
GRPC_PORT=50051

# 第三方可执行文件路径（留空则自动检测 PATH）
MEDIAMTX_BIN_PATH=tools/third_party/mediamtx/mediamtx.exe
MEDIAMTX_SRT_PUBLIC_HOST=192.168.1.100
MEDIAMTX_SRT_READ_HOST=192.168.1.100
VITE_BACKEND_TARGET=http://127.0.0.1:8000
```

> **注意**：`DJANGO_DEBUG=True` 时启动器 GUI 和播放窗口均显示标题栏，播放窗口不全屏，便于开发调试。
> `MEDIAMTX_SRT_PUBLIC_HOST` 用于生成给 OBS / 外部设备使用的局域网 SRT 推流地址；`MEDIAMTX_SRT_READ_HOST` 用于生成局域网读取地址；留空时系统会自动推断当前主机局域网 IP。
> `VITE_BACKEND_TARGET` 用于前端直连 Django 后端；前端开发服务器不再通过 Vite 反向代理 `/api/`。若从手机或其它局域网设备访问前端，且配置值仍为 `127.0.0.1`，前端会自动把后端主机替换为当前访问的局域网主机名。

### 2.5 安装第三方工具

**MediaMTX**（SRT 流服务）：

从 [MediaMTX Releases](https://github.com/bluenviron/mediamtx/releases) 手动下载，解压到 `tools/third_party/mediamtx/`。

MediaMTX 负责接收 OBS 通过 SRT 推送的流，并为播放器提供稳定的 SRT 读取入口。

**VLC/libVLC**（SRT 播放运行时）：

```powershell
.\tools\download_third_party.ps1
```

脚本会下载 Windows x64 VLC zip 包并解压到 `tools/third_party/vlc/runtime/`；如果系统已安装 VLC，播放器也会自动尝试使用系统安装目录。

### 2.6 初始化数据库

```powershell
uv run python manage.py migrate
```

### 2.7 安装前端依赖

```powershell
npm install --prefix frontend
```

## 3. 启动系统

### 3.1 一键启动（推荐）

使用 `runall` 命令在单个终端中同时启动 MediaMTX、Django 开发服务器、Vue 前端和 PySide6 播放器：

```powershell
# 默认启动（后端 0.0.0.0:8000，前端 0.0.0.0:5173）
uv run python manage.py runall

# 自定义后端监听地址和端口
uv run python manage.py runall --backend-host 0.0.0.0 --backend-port 8080

# 如需限制前端仅本机访问，可显式指定前端监听地址
uv run python manage.py runall --frontend-host 127.0.0.1

# 跳过前端或播放器（调试用）
uv run python manage.py runall --skip-frontend
uv run python manage.py runall --skip-player

# 跳过 MediaMTX（已在外部启动时使用）
uv run python manage.py runall --skip-mediamtx

# 自定义播放器轮询间隔
uv run python manage.py runall --poll-interval 0.2
```

启动流程：

1. **MediaMTX** 以子进程启动（提供 SRT 推流接收与 SRT 读取服务）
2. **Django 开发服务器** 以子进程启动（提供 REST + gRPC 服务）
3. **Vue 前端** 以子进程启动（提供浏览器控制台）
4. **PySide6 播放器** 以子进程启动并显示启动器 GUI
5. 多显卡主机可在启动器 GUI 中选择 SRT 渲染显卡，随后为播放窗口选择目标屏幕并创建播放窗口
6. 在 runall 终端按 Ctrl+C 时自动停止所有子进程

> **注意**：播放器是否全屏无边框由 `.env` 中的 `DJANGO_DEBUG` 开关控制；播放窗口屏幕分配和 SRT 渲染显卡由启动器 GUI 选择，之后通过 Web 控制台完成全部播放控制。

### 3.2 分进程启动（调试用）

开发调试时可分别启动各服务：

**终端 1：Django 服务端**

```powershell
uv run python manage.py runserver
```

服务端默认监听 `http://127.0.0.1:8000/`，同时在端口 `50051` 提供 gRPC 服务。

**终端 2：Vue 前端**

```powershell
npm --prefix frontend run dev
```

前端默认监听 `0.0.0.0:5173`，本机可访问 `http://127.0.0.1:5173/`，手机或其他局域网设备可访问 `http://<本机局域网IP>:5173/`。前端通过 `frontend/.env` 的 `VITE_BACKEND_TARGET` 直接访问后端 `/api/` 与 `/media/`，不再依赖 Vite 反代。

**终端 3：播放器**

```powershell
uv run python manage.py run_player
```

启动后显示启动器 GUI，多显卡主机可选择 SRT 渲染显卡，用户选择播放窗口屏幕后创建播放窗口；本地控制面板已移除，请使用 Web 控制台控制播放。

**终端 4：MediaMTX（可选）**

```powershell
.\tools\third_party\mediamtx\mediamtx.exe
```

> **注意**：MediaMTX 中已启用 SRT 推流与 SRT 读取，系统使用 SRT 接收推流，并由 libVLC 直接拉取 SRT 内容播放。

## 4. Web 控制台使用

浏览器打开 `http://127.0.0.1:5173/` 进入 Vue 控制台；手机访问时使用 `http://<本机局域网IP>:5173/`。Django 的 `http://127.0.0.1:8000/` 仅作为后端服务提示和 API 入口。

### 4.1 页面结构

控制台采用总览 + 独立显示控制页布局：

| 页面 | 功能 |
|------|------|
| 总览 | 查看在线源、活跃窗口、设备状态、预案数量和大屏模式 |
| 大屏左 / 大屏右 / TV 左 / TV 右 | 面向单个输出窗口的源切换、临时上传、关闭显示和源类型控制 |
| 媒体源 | 文件夹管理、上传、下载、移动、本地路径、网页源和直播源管理 |
| 播放控制 | 四窗口选择、播放/暂停/停止、PPT 翻页、视频 Seek、窗口音量和大屏模式 |
| 系统设置 | 系统音量、显示器目标、设备电源指令和窗口状态 |
| 预案管理 | 四窗口预案矩阵，支持创建、编辑、删除、置顶、激活和导入当前状态 |
| 关于 | 使用流程、关键入口、运行约束和验证命令 |

顶部工具栏提供全局操作：页面导航、停止当前窗口、SSE 连接状态指示。`single` 模式下大屏右入口会隐藏；切换到 `double` 后显示。

### 4.2 添加媒体源

**上传文件**：在「媒体源」页或任一显示控制页选择文件并上传，支持 PPT、视频、音频、图片等格式。上传时会显示进度条，系统根据文件扩展名自动检测源类型。

**临时上传即开**：在显示控制页选择文件后点击「临时上传即开」，该源会打开到当前显示窗口；播放器切换或关闭该源后自动清理临时源记录和上传文件。

**添加本地路径**：填写本机文件绝对路径（如 `D:\Videos\demo.mp4`），适用于大文件无需上传的场景。

**SRT 流**：外部设备通过 SRT 协议向 `srt://<主机IP>:8890?streamid=publish:<流标识>&latency=30000` 推流后，系统自动发现并注册到源列表中。也可手动点击刷新按钮立即同步流状态。

**网页源**：填写网页地址即可添加。未填写 `http://` 或 `https://` 时，系统默认按 `http://` 处理，适合局域网看板地址。

### 4.3 播放与控制

1. 在「大屏左 / 大屏右 / TV 左 / TV 右」页面选择媒体源，点击「打开源」开始播放
2. 在显示控制页查看当前状态，使用播放/暂停/停止按钮控制
3. PPT 源：使用上一页/下一页按钮翻页，或输入页码直接跳转；`.pptx` / `.ppsx` 会尝试解析提词器、页面媒体清单和 PNG 预览
4. PPT 打开后可进入专注模式，专注模式返回时会直接回到对应窗口的显示控制页
5. 视频源：使用播放/暂停/停止、进度条、前进/后退 10 秒、循环、窗口音量和静音控制
6. 图片和网页源：显示当前资源地址，支持切换源和关闭显示
7. 直播源：显示直播读取地址和播放状态，离线超过 1 小时的源会自动从源列表清理

手机访问时可直接进入对应显示控制页或 PPT 专注模式；专注模式保留大号「下一页 / 上一页」按钮，适合巡场时单手操作。

### 4.4 PPT 专注模式

PPT 源打开后，可从显示控制页进入「PPT 专注模式」。该页面隐藏顶部导航，只保留返回按钮、大号翻页按钮、媒体播放/暂停、当前页预览、下一页预览、页码状态和提词器文本。预览、提词器和媒体标记来自 `/api/sources/{source_id}/ppt-resources/` 保存的 PPT 解析资源；SSE 会轮询播放器写入的数据库状态，确保当前页和下一页预览随放映进度实时切换。当前页若有媒体对象，媒体播放/暂停按钮才可用。

### 4.5 切换显示目标和大屏模式

- **大屏 single**：只显示大屏左（窗口 1），窗口 2 静音并隐藏入口
- **大屏 double**：大屏左/右分别对应窗口 1/2，窗口 1/2 不静音
- **TV 左 / TV 右**：分别对应窗口 3/4，始终静音
- **显示器目标**：在「系统设置」页选择目标显示器并应用到当前窗口

### 4.6 设备和音量

「首页」和「系统设置」页提供拼接屏、电视左、电视右三类电源按钮，不显示也不保存开关机状态。拼接屏提供开机和关机按钮，通过 TCP 向 `192.168.5.10:8889` 发送两段控制帧；电视左和电视右各提供一个开关切换按钮，分别向 `192.168.5.161:8889`、`192.168.5.162:8889` 发送切换帧。所有电源指令均不读取设备返回。系统音量优先调用 Windows Core Audio 同步主音量和静音；若当前环境无法访问系统音频设备，则回退到运行态存储并在响应中标记 `system_synced=false`。

### 4.7 关闭播放

点击顶部工具栏的「停止当前窗口」按钮，或在显示控制页点击「关闭显示」。

## 5. gRPC 接口

外部系统（如中控系统）可通过 gRPC 远程控制播放。Vue 前端不再调用 gRPC，主通信通道为 REST API。

### 5.1 连接信息

- **地址**：`127.0.0.1:50051`（可通过 `GRPC_PORT` 配置）
- **服务名**：`scp_cv.v1.PlaybackControlService`
- **合同文件**：`protos/scp_cv/v1/control.proto`

### 5.2 可用方法

| 方法 | 说明 |
|------|------|
| `OpenSource` | 打开媒体源（参数：`media_source_id`, `autoplay`） |
| `CloseSource` | 关闭当前播放的源 |
| `ControlPlayback` | 播放控制（参数：`action` = PLAY/PAUSE/STOP） |
| `NavigateContent` | PPT 翻页 / 视频跳转（参数：`action`, `target_index`, `position_ms`） |
| `GetRuntimeStatus` | 获取运行时状态概览 |
| `GetPlaybackState` | 获取详细播放状态（含 PPT 页码、视频进度） |
| `ListDisplayTargets` | 列出可用显示器和拼接标签 |
| `SelectDisplayTarget` | 切换显示目标（参数：`display_mode`, `target_label`） |
| `WatchPlaybackState` | 订阅播放状态流 |
| `StopCurrentContent` | 停止当前播放（兼容旧接口，等价于 CloseSource） |

### 5.3 调用示例（Python）

```python
import grpc
from scp_cv.grpc_generated.scp_cv.v1 import control_pb2, control_pb2_grpc

# 建立连接
channel = grpc.insecure_channel("127.0.0.1:50051")
stub = control_pb2_grpc.PlaybackControlServiceStub(channel)

# 获取运行时状态
status = stub.GetRuntimeStatus(control_pb2.EmptyRequest())
print(status.source_type, status.playback_state)

# 打开媒体源（按 MediaSource ID）
reply = stub.OpenSource(control_pb2.OpenSourceRequest(
    media_source_id=1,
    autoplay=True,
))
print(reply.success, reply.message)

# PPT 翻下一页
reply = stub.NavigateContent(control_pb2.NavigateContentRequest(
    action=control_pb2.NAV_NEXT,
))

# 视频跳转到 10 秒
reply = stub.NavigateContent(control_pb2.NavigateContentRequest(
    action=control_pb2.NAV_SEEK,
    position_ms=10000,
))

# 关闭当前播放
reply = stub.CloseSource(control_pb2.CloseSourceRequest())
```

## 6. HTTP API 参考

所有 POST 接口需携带 CSRF Token（Cookie 中的 `csrftoken`）。
完整接口文档参见 `docs/API.md`。

### 源管理

| 方法 | 路径 | 参数 | 返回 |
|------|------|------|------|
| GET | `/api/folders/` | — | JSON `{success, folders}` |
| POST | `/api/folders/` | `name`, `parent_id`（可选） | JSON `{success, folder}` |
| GET | `/api/sources/` | `source_type`, `folder_id`（可选过滤） | JSON `{success, sources}` |
| POST | `/api/sources/upload/` | `file`, `name`, `source_type`, `folder_id`, `is_temporary` | JSON `{success, source}` |
| POST | `/api/sources/local/` | `path`, `name`, `source_type`, `folder_id` | JSON `{success, source}` |
| POST | `/api/sources/web/` | `url`, `name`, `folder_id` | JSON `{success, source}` |
| PATCH | `/api/sources/<source_id>/move/` | `folder_id` | JSON `{success, source}` |
| GET | `/api/sources/<source_id>/download/` | — | 文件下载 |
| GET/PUT | `/api/sources/<source_id>/ppt-resources/` | `resources`（PUT） | JSON `{success, resources}` |
| DELETE | `/api/sources/<source_id>/` | — | JSON `{success}` |

### 播放控制

> 路径中 `<window_id>` 为目标窗口编号（1-4）。

| 方法 | 路径 | 参数 | 返回 |
|------|------|------|------|
| POST | `/api/playback/<window_id>/open/` | `source_id`, `autoplay`（默认 true） | JSON `{success, sessions}` |
| POST | `/api/playback/<window_id>/control/` | `action`（play/pause/stop） | JSON `{success, sessions}` |
| POST | `/api/playback/<window_id>/navigate/` | `action`（next/prev/goto/seek）, `target_index`, `position_ms` | JSON `{success, sessions}` |
| POST | `/api/playback/<window_id>/ppt-media/` | `action`, `media_id`, `media_index` | JSON `{success, sessions}` |
| POST | `/api/playback/<window_id>/close/` | — | JSON `{success, sessions}` |
| PATCH | `/api/playback/<window_id>/loop/` | `enabled`（true/false） | JSON `{success, sessions}` |
| PATCH | `/api/playback/<window_id>/volume/` | `volume`（0-100） | JSON `{success, sessions}` |
| PATCH | `/api/playback/<window_id>/mute/` | `muted` | JSON `{success, sessions}` |

### 窗口标识

| 方法 | 路径 | 参数 | 返回 |
|------|------|------|------|
| POST | `/api/playback/show-ids/` | — | JSON `{success, sessions}` |

### 状态查询 & SSE

| 方法 | 路径 | 参数 | 返回 |
|------|------|------|------|
| GET | `/api/sessions/` | — | JSON `{success, sessions}` |
| GET | `/api/runtime/` | — | JSON `{success, runtime}` |
| PATCH | `/api/runtime/` | `big_screen_mode` | JSON `{success, runtime, sessions}` |
| GET | `/api/events/` | `last_id`（可选） | SSE 事件流 |

### SSE 事件类型

| 事件名 | 载荷 | 说明 |
|--------|------|------|
| `playback_state` | 会话快照 JSON | 播放状态变更时推送 |
| `heartbeat` | 序列号 | 每 30 秒发送一次保活 |

## 8. 常见问题

### 播放窗口没有显示内容

1. 确认 Django 服务端已启动并可正常访问 `http://127.0.0.1:8000/`
2. 确认播放窗口进程正在运行
3. 在 Web 控制台检查会话状态是否为「播放中」
4. 检查显示器选择是否正确（目标显示器名称需匹配系统检测结果）

### SRT 流无法播放

1. 确认 MediaMTX 已启动（检查 `http://127.0.0.1:9997/v3/paths/list`）
2. 确认 OBS 推流端 SRT URL 格式正确：`srt://<IP>:8890?streamid=publish:<流标识>&latency=30000`
3. OBS 设置：服务器填 SRT URL，流密钥留空
4. 在控制台确认流状态为「在线」后再打开

### 拼接模式不可用

系统需要检测到至少两台显示器才能启用左右拼接模式。确认：

1. 操作系统已正确识别多台显示器（设置 → 显示）
2. 显示器处于「扩展」模式而非「复制」模式
