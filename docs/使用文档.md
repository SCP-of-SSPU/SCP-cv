# SCP-cv 使用文档

本文面向 28#108 现场部署、值守和调试人员，说明 SCP-cv 的安装、配置、启动、播控流程和常见故障处理方式。

## 1. 系统说明

SCP-cv 用于控制上海第二工业大学 28#108 多媒体显示系统。系统由四类进程协作：

| 进程 | 作用 |
|------|------|
| Vue 控制台 | 浏览器播控界面，运行在 `frontend/` |
| Django 服务端 | REST、SSE、gRPC、媒体文件、播放会话状态 |
| PySide6 播放器 | 本机播放窗口，负责 PPT、视频、图片、网页和 SRT 显示 |
| MediaMTX | SRT 推流接收与读取服务 |

播放控制统一在 Web 控制台完成。本机播放器只负责创建输出窗口并执行后端下发的播放指令。

## 2. 环境准备

### 2.1 基础软件

- Windows 10/11
- Python 3.12+
- Node.js 20+
- Microsoft PowerPoint
- VLC/libVLC Windows x64 runtime
- MediaMTX Windows x64

### 2.2 安装依赖

```powershell
cd E:\Projects\SCP-cv

uv python install
uv sync

npm ci --prefix frontend
```

项目依赖以 `pyproject.toml`、`uv.lock`、`frontend/package.json` 和 `frontend/package-lock.json` 为准，不再维护 `requirements*.txt`。

### 2.3 准备第三方运行时

MediaMTX：

```text
tools/third_party/mediamtx/mediamtx.exe
tools/third_party/mediamtx/mediamtx.yml
```

VLC/libVLC 满足以下任一条件即可：

- 系统安装 Windows x64 VLC，默认路径 `C:\Program Files\VideoLAN\VLC`
- 将 Windows x64 VLC zip 包解压到 `tools/third_party/vlc/runtime/`

SRT 播放至少需要：

```text
tools/third_party/vlc/runtime/libvlc.dll
tools/third_party/vlc/runtime/libvlccore.dll
tools/third_party/vlc/runtime/plugins/access/libaccess_srt_plugin.dll
```

### 2.4 配置环境变量

复制模板：

```powershell
copy .env.example .env
copy frontend\.env.example frontend\.env
```

根目录 `.env` 管理后端和运行时：

```env
DJANGO_SECRET_KEY=please-change-me-for-production
DJANGO_DEBUG=True
DJANGO_ALLOWED_HOSTS=*
GRPC_PORT=50051
MEDIAMTX_BIN_PATH=E:\Projects\SCP-cv\tools\third_party\mediamtx\mediamtx.exe
MEDIAMTX_API_BASE=http://127.0.0.1:9997
MEDIAMTX_SRT_PORT=8890
MEDIAMTX_SRT_PUBLIC_HOST=
MEDIAMTX_SRT_READ_HOST=
```

`frontend/.env` 管理 Vite 控制台：

```env
VITE_FRONTEND_PORT=5173
VITE_BACKEND_TARGET=http://127.0.0.1:8000
```

现场局域网访问时，`VITE_BACKEND_TARGET` 必须写成浏览器可直连的后端地址，例如 `http://192.168.1.100:8000`。`runall` 会移除父进程继承的 `VITE_*` 变量，避免根目录 `.env` 或系统环境变量遮蔽 `frontend/.env`。

### 2.5 初始化数据库

```powershell
uv run python manage.py migrate
```

## 3. 启动系统

### 3.1 一键启动

```powershell
uv run python manage.py runall
```

常用参数：

```powershell
# 局域网访问
uv run python manage.py runall --backend-host 0.0.0.0 --frontend-host 0.0.0.0

# 跳过已外部启动的 MediaMTX
uv run python manage.py runall --skip-mediamtx

# 调试时跳过播放器或前端
uv run python manage.py runall --skip-player
uv run python manage.py runall --skip-frontend

# 自定义前端端口；未指定时读取 frontend/.env
uv run python manage.py runall --frontend-port 5174

# 纯命令行模式：跳过播放器启动器 GUI，直接按显示器 ID 创建 4 个窗口
uv run python manage.py runall --headless

# 后台模式：脱离当前终端生命周期，输出写入 logs/runall-service.log
uv run python manage.py runall --headless --service

# 指定窗口到 Windows 显示器 ID，并指定 GPU ID
uv run python manage.py runall --headless --window1 1 --window2 2 --window3 3 --window4 4 --gpu 0
```

`--headless` 会自动启动 MediaMTX、Django、Vue 前端和 PySide6 播放器，不再弹出显示器选择 GUI。未显式指定 `--window1`、`--window2`、`--window3`、`--window4` 时，窗口 1-4 默认映射到 Windows 显示器 ID 1-4；`--window3` 兼容别名 `--windows3`。未显式指定 `--gpu` 时，播放器不强制选择显卡，使用系统默认 GPU。

启动完成后：

1. 浏览器打开 `http://127.0.0.1:5173/`。
2. 手机或其它局域网设备打开 `http://<主机局域网IP>:5173/`。
3. 在播放器启动器中选择 SRT 渲染显卡和显示目标；若使用 `--headless`，则按命令行参数或默认显示器 ID 自动创建窗口。
4. 在 Web 控制台添加或选择媒体源并打开到指定窗口。

### 3.2 分进程启动

```powershell
# 终端 1：Django
uv run python manage.py runserver

# 终端 2：Vue
npm --prefix frontend run dev

# 终端 3：播放器
uv run python manage.py run_player

# 终端 3：播放器无 GUI 启动
uv run python manage.py run_player --headless --window1 1 --window2 2 --window3 3 --window4 4

# 终端 4：MediaMTX
.\tools\third_party\mediamtx\mediamtx.exe .\tools\third_party\mediamtx\mediamtx.yml
```

## 4. 控制台操作

### 4.1 页面入口

| 页面 | 用途 |
|------|------|
| 总览 | 查看系统状态、大屏模式、系统音量、设备电源 |
| 大屏左 / 大屏右 / TV 左 / TV 右 | 对指定窗口打开源、关闭显示、控制播放 |
| 媒体源 | 上传文件、添加网页源、本机路径和直播源，管理源列表 |
| 播放控制 | 四窗口播放、PPT 翻页、视频 Seek、窗口音量和循环 |
| 预案管理 | 保存四窗口组合，在列表中直接调用、置顶或取消置顶，一键恢复现场场景 |
| 系统设置 | 显示器目标、设备电源、运行态和开发信息 |

### 4.2 添加媒体源

- 上传文件：支持 PPT、视频、图片等常用格式。
- 临时上传即开：上传后直接打开到当前窗口，关闭或切换后清理临时源。
- 本机路径：适合不需要上传的大文件，例如 `D:\Videos\demo.mp4`。
- 网页源：填写 URL，可开启预热以减少现场切换等待。
- SRT 直播：外部设备推流到 `srt://<主机IP>:8890?streamid=publish:<流标识>&latency=30000` 后，系统会同步为直播源。

### 4.3 播放与控制

1. 进入对应显示控制页。
2. 选择媒体源，点击打开源。
3. 按源类型使用对应控制：
   - PPT：上一页、下一页、跳页、页面媒体播放/暂停、专注模式；专注模式横屏下支持左侧缩略图跳页。
   - 视频：播放、暂停、停止、进度条、循环、窗口音量。
   - 图片/网页：打开、关闭、切换源。
   - 直播：查看在线状态和拉流地址，必要时重新打开源。
4. 关闭显示时，后端会立即把窗口状态重置为空闲，播放器随后执行关闭指令。

### 4.4 大屏模式

- `single`：只使用大屏左，窗口 2 自动隐藏并静音。
- `double`：大屏左和大屏右分别对应窗口 1/2。
- TV 左 / TV 右：分别对应窗口 3/4，默认静音。

显示器目标在系统设置页应用。现场调整 Windows 显示设置后，建议重新启动播放器。

### 4.5 设备电源和音量

控制台提供拼接屏、TV 左、TV 右电源按钮。设备指令通过 TCP 发送，不读取设备返回状态。系统音量优先调用 Windows Core Audio；若当前环境无法同步系统音频设备，接口会返回 `system_synced=false`。

## 5. 接口和集成

REST API 的机器可读合同位于：

```text
docs/openapi.yaml
```

gRPC 合同位于：

```text
protos/scp_cv/v1/control.proto
```

gRPC 的 `PlaybackControlService` 保留给中控系统和自动化脚本使用，覆盖播放控制、状态查询、显示器查询、预案管理和媒体源管理。媒体源列表与新增/更新响应中的 `SourceItem` 字段与 REST 媒体源 payload 对齐，包含 `metadata`、文件属性、`keep_alive`、`preheat_enabled`、`preview_url`、`thumbnail_url`、`preview_kind` 和 `preview_label`。`UpdateSource` 仅允许更新显示名称、网页源 URL 和网页预热开关；文件型源路径仍通过重新上传或重新注册管理。

`GetRuntimeStatus.grpc_endpoint` 返回客户端可直接连接的地址。若服务监听配置为 `0.0.0.0` 或 `::`，状态响应会展示为 `127.0.0.1:<GRPC_PORT>`，避免外部脚本拿到不可连接的 wildcard 地址。

默认服务信息：

| 服务 | 地址 |
|------|------|
| REST / SSE | `http://127.0.0.1:8000/` |
| gRPC | `127.0.0.1:50051` |
| MediaMTX API | `http://127.0.0.1:9997/` |
| SRT | `srt://127.0.0.1:8890` |

## 7. 常见问题

### 7.1 前端 `.env` 修改后不生效

确认修改的是 `frontend/.env`，不是根目录 `.env`。修改后重启 Vite 或 `runall`。`runall` 已在启动前端时清理父进程中的 `VITE_*`，避免后端 `.env` 读入的变量覆盖前端配置。

### 7.2 手机能打开前端但接口失败

把 `frontend/.env` 中的 `VITE_BACKEND_TARGET` 改为手机可访问的后端地址，例如 `http://192.168.1.100:8000`，并用 `--backend-host 0.0.0.0` 启动后端。

### 7.3 SRT 直播无法播放

1. 打开 `http://127.0.0.1:9997/v3/paths/list` 检查 MediaMTX。
2. 检查 OBS 推流 URL 是否为 `srt://<主机IP>:8890?streamid=publish:<流标识>&latency=30000`。
3. 检查 VLC runtime 是否包含 `libvlccore.dll` 和 `plugins/access/libaccess_srt_plugin.dll`。
4. 查看 `logs/scp-cv.log` 与 `logs/mediamtx.log`。

### 7.4 PPT 不能播放

确认本机安装 Microsoft PowerPoint，且当前 Windows 用户可以正常打开 PPT 文件。PPT 自动化依赖 COM，服务器无桌面会话或权限不足时无法工作。播放器会使用 PowerPoint 窗口化放映并嵌入到各自播放窗口，支持多个窗口同时播放不同 PPT；如果仍出现第二个 PPT 顶掉第一个，先确认 PowerPoint 未被外部策略强制为全屏演讲者模式，再查看 `logs/scp-cv.log` 中的 PowerPoint HWND 查找日志。

### 7.5 `--service` 启动后如何查看状态

`--service` 会立即返回当前终端，实际服务输出写入 `logs/runall-service.log`，各子进程日志仍写入 `logs/mediamtx.log`、`logs/django.log`、`logs/vue-前端.log`、`logs/pyside-播放器.log` 等对应文件。需要停止后台服务时，可在 Web 控制台触发系统关闭，或在 Windows 任务管理器中结束后台 `python.exe` 及其子进程。

### 7.6 拼接模式不可用

确认 Windows 已识别至少两台显示器，显示模式为“扩展”而不是“复制”。调整显示器后重启播放器。
