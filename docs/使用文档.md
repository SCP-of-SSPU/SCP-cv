# SCP-cv 使用文档

## 1. 系统概述

SCP-cv 是一套基于 Django + PySide6 的单机大屏内容播控系统，能够在本地主机上完成：

- PPT/PPTX 演示文稿的上传、解析、逐页渲染与翻页控制
- SRT 视频流的接收与全屏播放（通过 MediaMTX）
- 单屏或左右拼接双屏显示
- 通过 Web 控制台（HTTP）和外部系统（gRPC）两种方式操控

系统由两个进程协作：**Django 服务端**（Web UI + API + gRPC）和 **PySide6 播放窗口**（全屏渲染），两者通过 SQLite 数据库同步状态。

## 2. 环境准备

### 2.1 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 |
| Python | 3.12 或更高版本 |
| 显示器 | 至少 1 台（拼接模式需 2 台） |

### 2.2 创建虚拟环境

```powershell
cd E:\Projects\SCP-cv
python -m venv .venv
.venv\Scripts\Activate.ps1
```

### 2.3 安装 Python 依赖

```powershell
# 开发环境（包含格式化、类型检查等工具）
pip install -r requirements-dev.txt

# 或仅安装运行时依赖
pip install -r requirements.txt
```

### 2.4 配置环境变量

复制 `.env.example` 为 `.env`，按需修改：

```env
# Django 基础配置
DJANGO_DEBUG=False
DJANGO_SECRET_KEY=请替换为随机密钥
DJANGO_ALLOWED_HOSTS=127.0.0.1,localhost

# gRPC 服务端口
GRPC_PORT=50051

# 第三方可执行文件路径（留空则自动检测 PATH）
MEDIAMTX_BIN_PATH=tools/third_party/mediamtx/mediamtx.exe
LIBREOFFICE_BIN_PATH=
```

> **注意**：`DJANGO_DEBUG=True` 时播放窗口会以窗口化模式运行（1280×720），便于开发调试。

### 2.5 安装第三方工具

**MediaMTX**（SRT 流服务）：

```powershell
.\tools\download_third_party.ps1 -DownloadMediaMTX
```

或从 [MediaMTX Releases](https://github.com/bluenviron/mediamtx/releases) 手动下载，解压到 `tools/third_party/mediamtx/`。

**LibreOffice**（PPT 转换引擎）：

```powershell
# 方式一：通过 winget 安装
.\tools\download_third_party.ps1 -InstallLibreOffice

# 方式二：手动安装
# 下载地址：https://www.libreoffice.org/download/
# 确保 soffice.exe 在 PATH 中，或在 .env 中配置 LIBREOFFICE_BIN_PATH
```

### 2.6 初始化数据库

```powershell
python manage.py migrate
```

## 3. 启动系统

系统需要启动两个进程，可选启动 MediaMTX（仅在需要 SRT 流时）。

### 3.1 启动 Django 服务端（终端 1）

```powershell
python manage.py runserver
```

服务端默认监听 `http://127.0.0.1:8000/`，同时在端口 `50051` 提供 gRPC 服务。

### 3.2 启动播放窗口（终端 2）

```powershell
# 正常模式：全屏、无边框、置顶
python manage.py run_player

# 调试模式：窗口化 1280×720，便于开发
python manage.py run_player --debug-window

# 自定义轮询间隔（秒）
python manage.py run_player --poll-interval 0.2
```

播放窗口每 0.5 秒（默认）轮询数据库，自动同步播放状态。

### 3.3 启动 MediaMTX（终端 3，可选）

```powershell
.\tools\third_party\mediamtx\mediamtx.exe
```

MediaMTX 不会被 Django 自动启动，需要手动运行。启动后：

- SRT 推流端口：`8890`
- RTSP 读取端口：`8554`
- 管理 API：`http://127.0.0.1:9997`

## 4. Web 控制台使用

浏览器打开 `http://127.0.0.1:8000/` 进入控制台。

### 4.1 页面结构

| 区域 | 功能 |
|------|------|
| 顶部工具栏 | 停止播放、刷新状态、SSE 连接状态 |
| 会话状态卡片 | 当前内容类型、播放状态、显示模式、页码进度 |
| 显示器选择 | 切换单屏/拼接模式，选择目标显示器 |
| PPT 控制卡片 | 上一页/下一页/跳转、当前页媒体列表 |
| SRT 控制卡片 | 流列表、打开/停止流 |
| 文件管理卡片 | 上传 PPT、资源列表、打开/删除操作 |

### 4.2 上传并播放 PPT

1. 在文件管理区域点击「选择文件」，选取 `.ppt` 或 `.pptx` 文件
2. 点击「上传」，系统自动完成：解析页数 → LibreOffice 转 PDF → 逐页渲染为 PNG
3. 解析完成后资源列表中状态变为「已就绪」
4. 点击资源行的「打开」按钮，PPT 第一页即刻显示在播放窗口
5. 使用 PPT 控制卡片的上一页/下一页/跳转按钮翻页

### 4.3 播放 SRT 流

1. 确保 MediaMTX 已启动
2. 外部设备通过 SRT 协议向 `srt://<主机IP>:8890?streamid=publish:<流标识>&pkt_size=1316` 推流
3. 系统自动发现并注册新流（每 5 秒轮询 MediaMTX），流会自动出现在 SRT 控制卡片中
4. 点击流右侧「打开」按钮，播放窗口切换为流播放模式
5. 也可手动点击「⟳ 刷新」按钮立即同步流状态

### 4.4 切换显示目标

- **单屏模式**：选择一台显示器，播放窗口全屏显示在该显示器上
- **左右拼接模式**：自动检测最左两台显示器，窗口横跨两屏

### 4.5 停止播放

点击顶部工具栏的「停止」按钮，或使用键盘 `Esc` 键（在播放窗口焦点时）。

## 5. gRPC 接口

外部系统（如中控系统）可通过 gRPC 远程控制播放。

### 5.1 连接信息

- **地址**：`127.0.0.1:50051`（可通过 `GRPC_PORT` 配置）
- **服务名**：`scp_cv.v1.PlaybackControlService`
- **合同文件**：`protos/scp_cv/v1/control.proto`

### 5.2 可用方法

| 方法 | 说明 |
|------|------|
| `GetRuntimeStatus` | 获取运行时状态（内容类型、播放状态、显示模式） |
| `ListDisplayTargets` | 列出可用显示器和拼接标签 |
| `OpenResource` | 打开 PPT 资源（参数：`resource_id`） |
| `ControlPptPage` | 翻页控制（参数：`action`=prev/next/goto, `page_number`） |
| `ControlCurrentMedia` | 媒体控制（参数：`media_id`, `action`=play/pause/stop） |
| `OpenStream` | 打开 SRT 流（参数：`stream_identifier`） |
| `StopCurrentContent` | 停止当前播放 |
| `SelectDisplayTarget` | 切换显示目标（参数：`display_mode`, `target_label`） |

### 5.3 调用示例（Python）

```python
import grpc
from scp_cv.grpc_generated.scp_cv.v1 import control_pb2, control_pb2_grpc

# 建立连接
channel = grpc.insecure_channel("127.0.0.1:50051")
stub = control_pb2_grpc.PlaybackControlServiceStub(channel)

# 获取运行时状态
status = stub.GetRuntimeStatus(control_pb2.RuntimeStatusRequest())
print(status.content_kind, status.playback_state)

# 打开 PPT 资源
reply = stub.OpenResource(control_pb2.OpenResourceRequest(resource_id="1"))
print(reply.success, reply.message)

# 翻到下一页
reply = stub.ControlPptPage(
    control_pb2.ControlPptPageRequest(action="next")
)

# 停止播放
reply = stub.StopCurrentContent(control_pb2.EmptyRequest())
```

## 6. HTTP API 参考

所有 POST 接口需携带 CSRF Token（Cookie 中的 `csrftoken`）。

| 方法 | 路径 | 参数 | 返回 |
|------|------|------|------|
| GET | `/` | — | 控制台 HTML 页面 |
| POST | `/upload/` | `file`（multipart 文件） | JSON `{success, message}` |
| POST | `/delete/` | `resource_id` | JSON `{success, message}` |
| POST | `/open-resource/` | `resource_id` | JSON `{success, message}` |
| POST | `/ppt-navigate/` | `direction`(prev/next/goto), `target_page` | JSON `{success, message}` |
| POST | `/stop/` | — | JSON `{success, message}` |
| POST | `/switch-display/` | `mode`(single/left_right_splice), `target_name` | JSON `{success, message}` |
| POST | `/open-stream/` | `stream_id` | JSON `{success, message}` |
| GET | `/api/session/` | — | JSON 会话状态快照 |
| GET | `/api/resources/` | — | JSON 资源列表 |
| GET | `/api/page-media/` | `resource_id`, `page` | JSON 当前页媒体列表 |
| GET | `/events/` | — | SSE 事件流 |

### SSE 事件类型

| 事件名 | 载荷 | 说明 |
|--------|------|------|
| `playback_state` | 会话状态 JSON | 播放状态变更时推送 |
| `resource_updated` | 资源信息 JSON | 资源上传/删除/状态变更时推送 |
| `heartbeat` | 序列号 | 每 30 秒发送一次保活 |

## 8. 常见问题

### PPT 上传后状态一直是「解析中」

检查 LibreOffice 是否正确安装：

```powershell
soffice --version
```

若未安装或不在 PATH 中，请在 `.env` 中配置 `LIBREOFFICE_BIN_PATH` 指向 `soffice.exe` 的完整路径。

### 播放窗口没有显示内容

1. 确认 Django 服务端已启动并可正常访问 `http://127.0.0.1:8000/`
2. 确认播放窗口进程正在运行
3. 在 Web 控制台检查会话状态是否为「播放中」
4. 检查显示器选择是否正确（目标显示器名称需匹配系统检测结果）

### SRT 流无法播放

1. 确认 MediaMTX 已启动（检查 `http://127.0.0.1:9997/v3/paths/list`）
2. 确认推流端 SRT URL 格式正确：`srt://<IP>:8890?streamid=publish:<流标识>&pkt_size=1316`（路径不含前导斜杠）
3. 在控制台确认流状态为「在线」后再打开

### 拼接模式不可用

系统需要检测到至少两台显示器才能启用左右拼接模式。确认：

1. 操作系统已正确识别多台显示器（设置 → 显示）
2. 显示器处于「扩展」模式而非「复制」模式
