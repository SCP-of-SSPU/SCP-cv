# 大屏内容播放系统需求文档（阶段一）

## 1. 文档信息

- **项目名称**：SCP-cv 大屏内容播放系统
- **阶段**：阶段一
- **文档版本**：v1.0
- **输出格式**：前后端融合的 Django 系统 + 本地播放器进程 + 外部 gRPC 接口
- **适用范围**：单台主机控制本机连接的显示器进行内容播放，支持 PPT / SRT / 文件管理

---

## 2. 项目目标

建设一套运行于**单台主机**上的内容播放系统，满足以下目标：

1. 支持上传、管理、删除和打开本地 PPT / PPTX 文件
2. 支持接收外部设备通过 **SRT** 推送的视频流并进行显示
3. 支持对 PPT 进行播放控制，包括：
   - 上一页
   - 下一页
   - 跳转指定页
   - 当前页内多个媒体的播放控制
4. 支持选择显示器进行显示
5. 将左右两个显示器**拼接为一个完整逻辑屏幕**
6. 播放时仅使用**一个显示窗口**
7. 显示窗口需满足：
   - 全屏
   - 无边框
   - 置顶
8. 在 **DEBUG 模式** 下：
   - 可仅使用一个显示器
   - 窗口无需置顶
9. 页面风格采用 **Fluent 2**
10. 不做任何鉴权，默认部署在可信任网络环境
11. Django 的本地 HTTP 服务与 gRPC 服务解耦：
   - **HTTP 仅供本系统页面使用**
   - **gRPC 仅供外部程序调用**
   - 浏览器页面不依赖 gRPC

---

## 3. 范围说明

## 3.1 本期范围

### 支持的内容类型
- PPT / PPTX
- SRT 推流
- 文件管理

### 支持的显示模式
- 单显示器全屏显示
- 指定显示器全屏显示
- 左右显示器拼接后作为一个完整逻辑显示区域进行播放

### 支持的控制能力
- PPT 打开
- PPT 翻页
- PPT 跳页
- 当前页媒体播放 / 暂停 / 停止
- SRT 流选择与打开
- 当前流断开
- 文件上传 / 打开 / 删除
- 显示器选择
- 单屏 / 拼接模式切换

## 3.2 本期不包含
- 左右显示器分别显示不同内容
- 视频文件播放
- 播放列表
- 多路 SRT 同时分别播放
- 用户登录、权限控制、鉴权
- 多主机集中调度
- 复杂预案编排系统
- 浏览器直接访问 gRPC

## 4. 系统定位

## 4.1 本系统职责
本系统负责：

- 本地资源管理
- PPT 解析与播放控制
- SRT 流显示
- 显示器选择与窗口投放
- 对外提供 gRPC 控制能力
- 提供本地操作页面

## 4.2 外部系统职责
外部已有项目负责：

- 通过 gRPC 调用本系统
- 远程触发播放或切换
- 查询本系统当前状态
- 将本系统纳入现有控制流程中

---

## 5. 运行模式

## 5.1 正常运行模式
用于正式展示环境，要求：

- 播放器窗口全屏
- 无边框
- 置顶
- 可绑定指定显示器
- 可选择左右显示器拼接模式
- 同一时刻仅有一个显示窗口

## 5.2 DEBUG 模式
用于开发与调试环境，要求：

- 可仅使用一个显示器
- 不强制置顶
- 可不要求拼接显示
- 仍保持单窗口显示逻辑

### DEBUG 模式判定
由 Django 的 DEBUG 开关控制。

---

## 6. 显示与窗口需求

## 6.1 显示器识别
系统应能识别当前主机已连接的显示器，并获取至少以下信息：

- 显示器编号
- 显示器名称
- 分辨率
- 相对位置
- 是否为主显示器

## 6.2 显示器选择
系统应支持以下显示目标选择：

- 使用单个显示器显示
- 使用左右两个显示器拼接显示

## 6.3 拼接显示
当选择“左右显示器拼接显示”时：

- 系统应将左右两个显示器视为一个完整的逻辑显示区域
- 播放窗口应跨越两个显示器
- 内容应作为一个整体进行显示
- 允许为适应拼接区域而进行内容拉伸或缩放
- 需明确当前拼接顺序为“左 + 右”

## 6.4 单窗口要求
无论使用单显示器还是拼接显示，系统都必须满足：

- 全程仅创建并使用**一个显示窗口**
- 不允许左屏一个窗口、右屏一个窗口的实现方式
- 窗口必须是：
  - 全屏
  - 无边框
  - 正常模式下置顶
  - DEBUG 模式下可不置顶

## 7. 业务流程

## 7.1 PPT 播放流程
1. 操作员上传或导入 PPT / PPTX 文件
2. 系统保存文件并解析基本信息
3. 系统将 PPT 转换为可播放资源
4. 操作员选择显示目标（单显示器或拼接）
5. 操作员打开指定 PPT
6. 本地播放器在指定显示区域内进行全屏显示
7. 操作员通过页面或外部 gRPC 控制翻页
8. 若当前页存在媒体，则可对当前页媒体进行播放控制

## 7.2 SRT 播放流程
1. 外部设备向系统指定 SRT 地址推流
2. SRT 服务接收流并注册流状态
3. 操作员选择显示目标
4. 操作员打开指定流
5. 本地播放器切换至流内容并全屏显示
6. 操作员可断开当前流或切换回 PPT

## 8. 功能需求

# 8.1 统一播放区域

系统阶段一仅维护**一个统一播放区域**。

要求：

- 同时仅播放一个内容源
- 内容源类型为：
  - PPT
  - SRT 流
- 同一时刻只存在一个显示窗口
- 播放内容根据目标显示器设置进行投放

---

# 8.2 PPT 功能

## 8.2.1 文件导入
系统应支持以下导入方式：

- 上传 `.ppt`
- 上传 `.pptx`
- 导入本地已有 PPT 文件路径

要求：
- 限制文件格式
- 保存原始文件
- 保存上传时间
- 保存最近使用时间
- 记录解析状态

## 8.2.2 PPT 基础信息
系统应对 PPT 保存以下信息：

- 文件名称
- 文件类型
- 文件路径
- 文件大小
- 页数
- 上传时间
- 最近使用时间
- 解析状态

## 8.2.3 PPT 打开
系统应支持打开指定 PPT 到当前播放区域。

要求：
- 默认从第一页打开
- 打开成功后应显示当前页码 / 总页数
- 当前播放状态需同步到页面和对外接口

## 8.2.4 翻页控制
系统应支持：

- 上一页
- 下一页
- 跳转到指定页

规则：
- 第一页继续上一页时保持第一页
- 最后一页继续下一页时保持最后一页
- 指定页越界时返回明确错误

## 8.2.5 当前页媒体识别
系统应支持识别当前页中的媒体对象，包括：

- 视频
- 音频

要求：
- 一页可包含多个媒体
- 系统需记录媒体列表
- 页面可展示当前页媒体清单

## 8.2.6 当前页媒体控制
系统应支持对当前页媒体逐个控制：

- 播放
- 暂停
- 停止

要求：
- 控制粒度到单个媒体
- 当前页切换后媒体列表同步刷新
- 当前页无媒体时应明确提示

---

# 8.3 SRT 流功能

## 8.3.1 SRT 接入
系统应支持接收外部设备通过 SRT 推送的视频流。

要求：
- 支持固定路径或标识配置
- 支持检测流是否在线
- 支持记录最近连接情况

## 8.3.2 流状态信息
系统应支持展示以下流信息：

- 流名称
- 流标识 / 路径 / streamid
- 在线状态
- 最近连接时间
- 当前是否正在播放

## 8.3.3 流播放控制
系统应支持：

- 打开指定流
- 停止当前流
- 从流切换到 PPT
- 从 PPT 切换到流

要求：
- 切换过程有状态提示
- 流断开时应提示异常
- 打开失败时应返回错误原因

---

# 8.4 文件管理

## 8.4.1 文件类型
阶段一文件管理仅覆盖：

- PPT / PPTX 原始文件
- 系统派生出的 PPT 播放资源
- SRT 流记录或流配置

## 8.4.2 文件上传
系统应支持文件上传，要求：

- 可选择本地文件
- 可显示上传结果
- 上传成功后自动加入资源列表

## 8.4.3 文件列表
系统应提供文件列表，至少展示：

- 名称
- 类型
- 大小
- 页数（PPT）
- 上传时间
- 最近使用时间
- 当前状态
- 操作项

## 8.4.4 文件删除
系统应支持删除文件。

要求：
- 删除前需确认
- 正在播放中的文件应先停止再删除，或禁止直接删除
- 删除后应清理关联派生资源

## 8.4.5 文件打开
资源列表中应支持直接打开文件到当前播放区域。

## 9. 页面需求

## 9.1 页面设计原则
- 页面数量尽可能少
- 页面风格采用 **Fluent 2**
- 优先使用：
  - 卡片
  - 抽屉
  - 弹窗
  - 顶部命令栏
- 不做复杂后台管理式信息层级

## 9.2 页面风格要求（Fluent 2）
页面整体设计风格应满足：

- 简洁、现代、平面化
- 使用柔和圆角
- 使用清晰分层的卡片
- 使用轻量阴影与边界
- 以中性色为主、强调色适度点缀
- 命令区、状态区、资源区结构清晰
- 大按钮适合触控和快速操作
- 操作反馈明确

## 9.3 页面结构建议

### 页面一：播放控制台
作为主页面，承担主要功能。

#### 顶部命令栏
包含：
- 当前内容类型
- 显示目标选择
- 打开资源
- 打开流
- 上传文件
- 停止当前内容
- 刷新状态

#### 主状态区
采用卡片显示：
- 当前播放内容
- 当前状态
- 当前显示模式
- 当前显示器/拼接信息
- 当前页码 / 总页数（PPT）
- 当前流在线状态（SRT）

#### PPT 控制卡
包含：
- 上一页
- 下一页
- 跳页输入
- 当前页媒体列表
- 每个媒体的播放 / 暂停 / 停止按钮

#### SRT 控制卡
包含：
- 流选择
- 流状态
- 打开流
- 停止流

#### 文件管理卡
包含：
- 文件列表
- 打开
- 删除
- 搜索（可选）

#### 上传弹窗 / 抽屉
用于：
- 上传 PPT
- 导入本地文件路径

### 页面二：资源页（可选）
若主页面过重，可将文件管理拆出为第二页。

## 10. 技术栈需求

## 10.1 总体技术路线
系统采用**前后端融合架构**，不采用前后端分离。

### 架构要求
- 页面由 Django 服务端渲染
- 浏览器仅通过 HTTP 与 Django 通信
- gRPC 不直接暴露给页面
- gRPC 仅用于外部程序调用
- HTTP 与 gRPC 服务逻辑解耦

---

## 10.2 技术栈要求

### 1. Web 与业务层
- **Python 3.12+**
- **Django**
- **Django Templates**
- **Django StaticFiles**
- **Django Form / CBV / Function View**

要求：
- 用于页面渲染
- 用于文件上传与管理
- 用于页面交互提交
- 不引入独立前端工程

### 2. 内部与外部控制接口层
- **grpcio**
- **grpcio-tools**
- **protobuf**
- **django-socio-grpc**

要求：
- gRPC 服务只给外部程序使用
- 与本地 HTTP 页面解耦
- 不要求浏览器支持 gRPC
- Django 页面操作可通过内部调用服务层，不强依赖 gRPC

### 3. 本地播放器层
- **PySide6**
- **Qt Widgets**
- **Qt Multimedia**
- 视情况使用 **Qt WebEngine**

要求：
- 创建唯一显示窗口
- 支持全屏、无边框、置顶
- 支持跨显示器窗口定位与拼接
- 支持 DEBUG 模式下的简化显示
- 支持媒体播放与状态反馈

### 4. PPT 处理层
- **python-pptx**
- **LibreOffice CLI**
- 可选：**PyMuPDF / Pillow** 用于页面图像处理

要求：
- 读取 PPT 基础信息
- 提取页数与媒体信息
- 将 PPT 转换为可显示资源
- 为播放器提供页面资源

### 5. SRT 媒体服务层
- **MediaMTX**

要求：
- 接收 SRT 推流
- 提供流状态
- 为播放器提供稳定读取入口
- 与 Django 解耦

### 6. 数据存储层
- **SQLite**

要求：
- 存储文件信息
- 存储播放状态
- 存储流记录
- 存储解析结果与资源索引

### 7. 任务与异步处理
- **不进行支持，不要引入其它框架**

---

## 11. HTTP 与 gRPC 解耦要求

## 11.1 HTTP 责任
HTTP 服务仅负责：

- 页面渲染
- 表单提交
- 文件上传
- 文件删除
- 状态展示
- 本地操作入口

HTTP 页面不要求、也不允许依赖浏览器 gRPC 能力。

## 11.2 gRPC 责任
gRPC 仅负责：

- 给外部程序调用
- 外部控制命令接入
- 外部状态查询
- 与本地播放器或服务层进行程序间控制

## 11.3 解耦要求
必须满足：

- 页面是否打开，不影响 gRPC 服务可用性
- gRPC 服务是否被外部调用，不影响本地 HTTP 页面使用
- HTTP 与 gRPC 为两个独立接入面
- 浏览器只走 HTTP
- 外部程序优先走 gRPC
